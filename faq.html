frequently asked questions
what is fileditch?
fileditch is a fast, anonymous, and minimal file hosting service. there are no registration pages, account setups, or usage payloads required. simply drop your files and copy the generated link.
what is the difference between permanent and temporary uploads?
permanent uploads (hosted via new.fileditch.com) remain active indefinitely, storage permitting. temporary uploads (hosted via temp.fileditch.com) are automatically deleted 72 hours after upload, which is ideal for quick, one-off transfers.
what are the file upload limits?
you can upload files up to 100 gb in size. most file formats are supported.
are there any banned file extensions?
yes. to prevent malware distribution, security-sensitive file formats are blocked. executables, installer packages, scripts, and system configurations are rejected:
blocked: .php, .html, .htm, .js, .exe, .apk, .sh, .py, .bat, .cmd, .msi, .vbs, .scr, .cpl
what is the "cors blocked" error and why does it happen?
cross-origin resource sharing (cors) is a browser security protocol. fileditch distribution servers do not send cors wildcard headers, which prevents client-side web apps from fetching files directly. we resolve this by default using your specified worker proxy:
proxy: api.pidi.workers.dev
for files larger than 50mb, the proxy connection may reset or time out due to cloudflare worker execution limits. in these cases, it is highly recommended to use a cors-unblocking browser extension to enable direct, high-speed streaming.
how long are my files hosted?
files uploaded to our permanent servers remain hosted indefinitely as long as they comply with our terms of service (no malware, spam, or copyright abuse) and receive download activity. files uploaded via the temporary api are auto-deleted after 72 hours.
is there an api available?
yes, fileditch provides programmatic upload endpoints. you can upload files using simple multipart POST requests:
curl -F "file=@yourfile.mp3" https://new.fileditch.com/upload.php
for temporary uploads, simply route the requests to:
curl -F "file=@yourfile.mp3" https://temp.fileditch.com/upload.php
