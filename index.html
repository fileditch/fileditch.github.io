Upload Methods

Multipart Form-Data (Recommended) Standard browser-compatible file upload method. It encodes the file as form fields. This is the most reliable, compatible, and highly recommended method for all browsers.
Raw Body Stream Sends the raw file payload directly in the HTTP request body (`application/octet-stream`) without any wrapper metadata. This matches direct `curl -T` behavior and is slightly faster for large files, but is less supported by legacy proxies.

Fileditch Streamer Info

How it works: Paste a Fileditch viewer link (e.g., `https://fileditchfiles.me/file.php?f=...`). The streamer fetches the HTML source of the page through a Worker proxy (`api.pidi.workers.dev`) to bypass browser CORS blocks, extracts the temporary direct download link (with security `md5` & `expires` tokens), and loads the direct stream in the browser.
Supported Filetypes for Streaming:
  • Audio: mp3, ogg, wav, m4a, flac, aac, opus, wma
  • Video: mp4, mkv, webm, mov, avi, m4v, 3gp
  • Image: png, jpg, jpeg, gif, webp, bmp, ico, tiff
  • Document: pdf, txt, log, json, md, csv
